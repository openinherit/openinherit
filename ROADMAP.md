# INHERIT Roadmap

## Current: v1.0 (March 2026)

- 18 core entity schemas (13 candidate, 5 draft)
- 5 common type schemas
- 13 jurisdiction/cultural extensions (all draft)
- OpenAPI 3.1 schema bundle + 109-endpoint Reference REST API
- TypeScript SDK + Python validation example
- Language-agnostic test suite
- Web validator + will-to-INHERIT scanner at openinherit.org

## Next: v1.1 (target Q3 2026)

- Promote first schemas to stable (requires two independent implementations)
- Level 3 conformance validation (extension-specific field checking)
- Additional language SDKs (Python, Go)
- SchemaStore listing for IDE autocompletion
- Interactive API explorer (Swagger UI)
- First external contributor PRs

## Future: v1.2+

- Formal specification document (W3C-style normative prose)
- Digital asset entity (cryptocurrency, online accounts)
- Reference REST API implementation (working server, not just spec)
- Multi-language OCR for will scanner
- Community extension template repository

## Long Term: v2.0

- Custom INHERIT vocabulary (formal JSON Schema vocabulary, not annotation-only)
- Breaking changes to draft schemas based on implementation feedback
- Multi-version support policy

## How to Influence the Roadmap

File an issue or proposal on GitHub. All roadmap decisions are discussed publicly.
Partnership firms get a steering committee seat — see [Becoming a Partner](docs/partners/becoming-a-partner.md).
